{% trans "معرفی فرم تخصیص قوانین" %}

{% trans "فرم 'مدیریت پست‌ها و قوانین دسترسی' به شما اجازه می‌دهد:" %}

  • {% trans "همه پست‌های سازمانی (مثل مدیر مالی تهران) را ببینید." %}
  • {% trans "سطح هر پست (مثل مدیر، کارشناس) را تغییر دهید." %}
  • {% trans "قوانین دسترسی (مثل تأیید فاکتور یا امضای پرداخت) را با تیک زدن به پست‌ها بدهید." %}
  • {% trans "برای هر قانون، مرحله گردش کار (مثل بررسی اولیه) را انتخاب کنید." %}
{% trans "دسترسی هلدینگ" %}: {% trans "اگر در هلدینگ هستید، می‌توانید همه پست‌ها و قوانین را مدیریت کنید، حتی برای شعبه‌ها و معاونت‌ها." %}

{% trans "نحوه ثبت قوانین با تیک زدن" %}

{% trans "برای تخصیص قوانین به پست‌ها، این مراحل را دنبال کنید:" %}

  1. {% trans "ورود به فرم" %}

    {% trans "وارد سیستم شوید و به صفحه 'لیست قوانین دسترسی' بروید. روی دکمه 'مدیریت پست‌ها و قوانین' کلیک کنید تا فرم باز شود (آدرس: /access-rules/assign/)." %}

  2. {% trans "مشاهده جدول پست‌ها" %}

    {% trans "در فرم، جدولی می‌بینید که هر ردیف یک پست است. ستون‌ها شامل:" %}

    • {% trans "پست: نام پست (مثل مدیر مالی تهران)." %}
    • {% trans "سازمان: مثل هلدینگ یا شعبه تهران." %}
    • {% trans "شاخه: مثل مالی یا بازرگانی." %}
    • {% trans "سطح: سطح پست (مثل 1 برای کارشناس، 5 برای مدیرکل)." %}
    • {% trans "قوانین دسترسی: گزینه‌هایی برای تخصیص قوانین." %}
  3. {% trans "تغییر سطح پست" %}

    {% trans "برای هر پست، از منوی کشویی در ستون 'سطح' یک عدد (مثل 1 تا 5) انتخاب کنید. مثلاً، برای مدیر شعبه سطح 3 و برای مدیرکل سطح 5 مناسب است." %}

    {% trans "هشدار" %}: {% trans "سطح‌ها برای مرتب‌سازی پست‌ها مهم‌اند. سطح‌های بالاتر (مثل 5) معمولاً دسترسی بیشتری دارند." %}
  4. {% trans "تیک زدن قوانین" %}

    {% trans "در ستون 'قوانین دسترسی'، برای هر پست قوانین موردنظر را تیک بزنید. مثلاً:" %}

    • {% trans "فاکتور - تأیید: برای پست‌هایی که باید فاکتور تأیید کنند (مثل مدیر مالی)." %}
    • {% trans "تنخواه - رد: برای پست‌هایی که می‌توانند تنخواه را رد کنند." %}
    • {% trans "دستور پرداخت - امضا: فقط برای مدیرکل یا پست‌های سطح بالا." %}
  5. {% trans "انتخاب مرحله" %}

    {% trans "برای هر قانونی که تیک زدید، از منوی کشویی کنارش یک مرحله گردش کار (مثل 'بررسی اولیه' یا 'تأیید نهایی') انتخاب کنید. این مشخص می‌کند قانون در چه مرحله‌ای اعمال شود." %}

    {% trans "هشدار" %}: {% trans "اگر قانونی را تیک زدید، حتماً مرحله را انتخاب کنید، وگرنه قانون ذخیره نمی‌شود." %}
  6. {% trans "ذخیره تغییرات" %}

    {% trans "بعد از تنظیم سطح‌ها و تیک زدن قوانین، دکمه 'ذخیره' را بزنید. قوانین در سیستم ثبت می‌شوند و می‌توانید آن‌ها را در 'لیست قوانین دسترسی' یا 'گزارش قوانین پست‌ها' ببینید." %}

  7. {% trans "بررسی گزارش" %}

    {% trans "برای دیدن اینکه هر پست چه قوانینی دارد، به صفحه 'گزارش قوانین پست‌ها' (/access-rules/report/) بروید. این صفحه لیستی از پست‌ها و قوانینشان (مثل تأیید فاکتور) را نشان می‌دهد." %}

{% trans "مثال کاربردی" %}

{% trans "فرض کنید می‌خواهید به پست 'مدیر مالی تهران' اجازه دهید فاکتورها را تأیید کند و به 'مدیرکل هلدینگ' اجازه دهید دستورات پرداخت را امضا کند:" %}

  1. {% trans "وارد فرم شوید." %}

  4. {% trans "در ردیف 'مدیرکل هلدینگ'، سطح را روی 5 تنظیم کنید." %}
  5. {% trans "تیک 'دستور پرداخت - امضا' را بزنید و مرحله 'تأیید نهایی' را انتخاب کنید." %}
  6. {% trans "دکمه 'ذخیره' را بزنید." %}
  7. {% trans "در گزارش قوانین، چک کنید که قوانین درست ثبت شده‌اند." %}
